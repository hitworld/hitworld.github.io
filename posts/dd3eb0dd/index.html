安装内核

编译内核

linux内核源代码下载:https://mirrors.tuna.tsinghua.edu.cn/kernel/.

对于kernel 3.18.2,需要修改Makefile将-O3改为-O1,这样编译出的内核会包含调试信息且能被gdb调试.

重点配置(有些配置可能因为版本不同而不一样).

  • 取消KernelHacking --> Write protect kernel read-only data structures.
  • 取消Processor type and features --> Paravirtualized guest support.
  • 选中KernelHacking --> Compile the kernel with debug info.
  • 选中KernelHacking --> Compile the kernel with frame pointers.
  • 全选KernelHacking --> KGDB:kernel debugging with remote gdb,其下的全部都选中.

安装符号文件

符号文件会被安装到/usr/lib/debug/目录下,最重要的kernel文件在boot子目录下.

安装源代码

启用deb-src源,然后sudo apt-get update更新.

源代码的压缩包会被放在/usr/src目录下的同名文件夹下.

busybox

用busybox来生成文件系统,编译配置(有些配置可能因为版本不同而不一样).

  • 设置Busybox Settings --> Build Options -> Build Busybox as a static binary.
  • 关闭Linux System Utilities --> Support mounting NFS file system.
  • 关闭Networking Utilities --> inetd

编译成x86.在如下两个选项中输入对应的值.

make install后,在busybox源代码目录下有一个_install目录存放编译后的文件.

配置target

/etc/grub.d/40_custom.

该文件可以添加grup启动选项,只需要常规选项中的内核命令行处增加KGDB选项:kgdbwait kgdb8250=io,03f8,ttyS0,115200,4 kgdboc=ttyS0,115200 kgdbcon nokaslr.

kgdbwait让系统启动时就进入调试模式,kgdboc是KGDB over console的缩写,ttyS0代表通过1号串口进行通信,波特率为115200.后面的nokaslr是禁止内核空间的地址随机化.
