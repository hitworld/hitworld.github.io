属性文法

属性文法也称为属性翻译文法,其以上下文无关文法为基础,为每个文法符号配备若干相关的值(也就是属性),代表与文法符号相关的信息,如:类型,值,代码序列,符号表内容等.

同时属性文法对于文法的每个产生式都配备了一组属性的语义规则,对属性进行计算和传递.

综合属性:用于自下而上的传递信息.

对于语法规则,根据右部候选式中的符号的属性计算左部被定义符号的综合属性;对于语法树,根据子结点的属性和父结点自身的属性计算父结点的综合属性.

继承属性:用于自上而下的传递信息.

对于语法规则,根据右部候选式中的符号的属性和左部被定义符号的属性计算右部候选式中的符号的继承属性;对于语法树,根据父结点和兄弟结点的属性计算子结点的继承属性.

所有语义规则都可以写成对某些属性进行函数变换,然后将变换结果设置为某个属性的值.

属性依赖

对于每个产生式A->α都有一套与之相关联的语义规则,每条规则的形式为:b := f(c1, c2, c3, ..., ck),属性b依赖于属性c1, c2, c3, ..., ck.

b是A的一个综合属性并且c1, c2, c3, ..., ck是产生式右边文法符号的属性.或者b是产生式右边某个文法符号的一个继承属性并且c1, c2, c3, ..., ck是A或产生式右边任意文法符号的属性.

终结符只有综合属性,由词法分析器提供.非终结符两者都可以有,文法开始符号的所有继承属性作为属性计算前的初始值.

语义规则所描述的工作可以包括属性计算,静态语义检查,符号表操作,代码生成等.

对出现在产生式右边的继承属性和出现在产生式左边的综合属性都必须提供一个计算规则.属性计算规则中只能使用相应产生式中的文法符号的属性.

出现在产生式左边的继承属性和出现在产生式右边的综合属性不由所给的产生式的属性计算规则进行计算,由其他产生式的属性计算规则计算或由属性计算器的参数提供.

带注释的语法树

在语法树中,一个结点的综合属性的值由其子结点和它本身的属性值所确定.使用自底向上的方法在每一个结点处使用语义规则计算综合属性的值.

在语法树中,一个结点的继承属性由其父结点,其兄弟结点和其本身的某些属性确定.用继承属性来表示程序设计语言结构中的上下文依赖关系很方便.

属性计算

基于属性文法的处理方法.

语义规则的计算包括:产生代码,在符号表中存放信息,给出错误信息,执行任何其他动作.

对输入串的翻译就是根据语义规则进行计算.

由源程序的语法结构所驱动的处理方法就是语法制导翻译法:输入串->语法树->按照语义规则计算属性.

有三种方法:依赖图,树遍历,一遍扫描.

依赖图

在一棵语法树中的结点的继承属性和综合属性之间的相互依赖关系可以由依赖图(有向图)描述.

为每一个包含过程调用的语义规则引入一个虚综合属性b,把语义规则写成b := f(c1, c2, c3, ..., ck)的形式.

依赖图为每一个属性设置一个结点,如果属性b依赖于属性c,则从属性c的结点画一条有向边到属性b的结点.

依赖图的构建算法.

如果一个属性文法不存在属性之间的循环依赖关系,则称该文法为良定义的.

一个依赖图的任何拓扑顺序都给出一个语法树中结点的语义规则计算的有效次序.

基础文法用于建立输入符号串的语法分析树,然后根据语义规则建立依赖图,再根据依赖图的拓扑排序,得到计算语义规则的顺序.

也就输输入串->语法树->依赖图->语义规则计算次序.

树遍历

树遍历的属性计算方法:假设语法树已经建立,且书中已带有开始符号的继承属性和终结符的综合属性,以某种次序遍历语法树,直至计算出所有属性,例如深度优先,从左到右的遍历.

树遍历算法.

一边扫描

一边扫描的处理方法:在语法分析的同时计算属性值,关键点在于所采用的语法分析方法和属性的计算次序.

语法制导翻译法就是为文法中每个产生式配上一组语义规则,并且再语法分析的同时执行这些语义规则.

对于自上而下分析,一个产生式匹配输入串成功时进行语义规则的计算;对于自下而上分析,一个产生式被用于进行归约时进行语义规则的计算.

抽象语法树是一种源程序的中间代码表示,它把语法树中那些对翻译不必要的信息去掉.

建立表达式的抽象语法树.

  • mknode(op, left, right):建立一个运算符号结点,标号是op,两个域left,right分别指向左子树和右子树.
  • mkleaf(id, entry):建立一个标识符结点,标号是id,域entry指向标识符在符号表中的入口.
  • mkleaf(num, val):建立一个数结点,标号是num,域val存放数的值.

示例:一个S-属性文法.

S-属性文法

仅使用综合属性的属性文法称为S-属性文法,S-属性文法适合一遍扫描的自下而上分析.

在自下而上的分析器分析输入符号串的同时计算综合属性:其中分析栈中保存语法符号和有关的综合属性值,每当进行归约时,新的语法符号的属性值就由栈中正在归约的产生式右边符号的属性值来计算.

L-属性文法

L-属性文法:对于每个产生式A->X1X2X3...Xn,其每个语义规则中的每个属性或者是综合属性,或者是Xi(1<=i<=n)的一个继承属性且这个继承属性仅依赖于A的继承属性或产生式Xi左边符号(X1,X2,...,Xi-1)的属性.

S-属性文法一定是L-属性文法.

L-属性文法适合一遍扫描的自上而下分析.

L-属性文法和自顶向下翻译:与LL(1)自上而下分析方法结合,深度优先建立语法树,按照语义规则计算属性.

翻译模式

语义规则:给出了属性计算的定义,没有属性计算的次序等实现细节.

翻译模式:给出使用语义规则进行计算的次序,把实现细节补充了.

在翻译模式中,和文法符号相关的属性和语义规则(也称语义动作)用花括号括起来,插入到产生式右部的合适位置上.

设计翻译模式的原则:必须保证当某个动作引用一个属性时该属性已被定义.

L-属性文法本身就能保证每个动作不会引用尚未被计算出来的属性.

建立翻译模式的过程.

当只需要综合属性时:为每一个语义规则建立一个包含赋值的动作,并把这个动作放在相应产生式右边的末尾.

如果既有综合属性又有继承属性,就要保证:产生式右部符号的继承属性必须在该符号以前的动作计算出来;一个动作不能引用该动作右边的符号的综合属性;产生式左边非终结符的综合属性只有在它所引用的所有属性都计算出来后才能计算,计算这种属性的动作通常可放在产生式右端的末尾.

翻译模式示例

广义上来讲,任何对信息的变换,从一种形式到另一种形式都能称之为翻译.

例如:数学格式语言EQN.

语义动作的执行时机统一

可以把所有的语义动作都放在产生式的末尾从而语义动作的执行时机统一.

对于本身不能放在产生式末尾的语义动作的转换方法:加入新的产生式M -> ε,把嵌入在产生式的每个语义动作用不同的非终结符M代替,并把这个动作放在产生式M -> ε的末尾.

消除翻译模式中的左递归

语义动作是在相同位置上的符号被展开时(匹配成功)执行的.

为了构造不带回溯的自顶向下语法分析,必须消除文法中的左递归.当消除一个翻译模式的基本文法的左递归时同时考虑属性计算,这适合带综合属性的翻译模式.

递归下降翻译器的设计

分析程序由一组递归子程序组成,每个非终结符对应一个子程序.

对每个非终结符A构造一个函数过程.

  • 继承属性:对A的每个继承属性设置为函数的一个形式参数.
  • 综合属性:实现为函数的返回值.若有多个综合属性,作为结构返回.
  • A的产生式中的每一个文法符号的每一个属性实现为局部变量.

函数功能:非终结符A对应的函数过程中,根据当前的输入符号决定使用哪个产生式候选.

函数代码按照产生式右部从左到右的顺序实现.

  • 对于带有综合属性x的终结符X,把x的值存入为X.x设置的变量中,然后产生一个匹配X的调用,并继续读入一个输入符号.
  • 对于每个非终结符B,产生一个右边带有函数调用的赋值语句c=B(b1,b2,...,bk),其中,b1,b2,...,bk是为B的继承属性设置的变量,c是为B的综合属性设置的变量.
  • 对于语义动作,把动作的代码抄进分析器中,用代表属性的变量来代替对属性的每一次引用.
