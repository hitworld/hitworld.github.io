IS-pwn-Linux-glibc-2.29-基础与攻击方式

发表于 2021-07-03 | 分类于 IS ， pwn | 热度： ℃
本文字数： 3.3k | 阅读时长 ≈ 3 分钟

变化

  • unsorted bin attack失效.
  • house_of_force失效,会检测size是否大于av->system_mem.
  • 对tcache dup进行检测,覆盖bk(通常是uaf,但这样也就没必要double free)或者修改size,或者使用house_of_botcake.
  • setcontext的参数改成了rdx.

large bin attack

off_by_one

主要原理就是利用残余在large bin上的fd_nextsize和bk_nextsize指针.

  • 首先malloc一个large bin,称作chunk_A,而fake_chunk就是chunk_A + 0x10,在chunk_A的bk位置上写好size,并部分覆盖chunk_A的fd_nextsize到一个可以控制其bk的chunk上,比如从small bin或者unsorted bin中拿出的chunk,如果其bin中有多个chunk的话,那么拿出来的chunk的bk上必定残留了heap指针,可以通过部分覆盖使其指向fake_chunk,以便绕过unlink检查.
  • 由于chunk_A的bk_nextsize并没有修改,所以其指向的是chunk_A本身,为了绕过unlink检查(p->fd->bk == p && p->bk->fd == p).需要将chunk_A的fd指向fake_chunk,这里可以利用fastbin的链表特性在chunk_A上写好heap地址,然后进行部分覆盖使其指向chunk_A + 0x10,则这样就能绕过glibc-2.29的检查.

ORW的方法

gadget + setcontext

gadget位置为getkeyserv_handle + 576.

这个gadget从2.29到2.32都可用.

gadget + 栈迁移

gadget位置为svcudp_reply + 26.

通过rdi控制rbp进而控制rax并执行跳转,只需要在rax + 0x28的位置设置leave；ret即可完成栈迁移.
